authoritative source. Any error in the gap file propagates directly into the gold standard, which is the training signal 
for the OAPI optimizer. Three independent bug categories were identified.

7.2  Recommended Post-Processing Validation 
The arithmetic rules (BIMS, PHQ-2) are safety-critical. Even with the corrected prompt, a post-processing validator 
in Python is strongly recommended as a backstop. The LLM output should be checked before saving. 

 
def validate_gap_answers(answers: dict) -> list[str]: 
    errors = [] 
    # BIMS two-level arithmetic 
    if all(k in answers for k in ['C0300A','C0300B','C0300C','C0300']): 
        expected = int(answers['C0300A']) + int(answers['C0300B']) + int(answers['C0300C']) 
        if int(answers['C0300']) != expected: 
            errors.append(f'C0300={answers["C0300"]} but sub-sum={expected}') 
    if all(k in answers for k in ['C0400A','C0400B','C0400C','C0400']): 
        expected = int(answers['C0400A']) + int(answers['C0400B']) + int(answers['C0400C']) 
        if int(answers['C0400']) != expected: 
            errors.append(f'C0400={answers["C0400"]} but sub-sum={expected}') 
    if all(k in answers for k in ['C0200','C0300','C0400','C0500']): 
        expected = int(answers['C0200']) + int(answers['C0300']) + int(answers['C0400']) 
        if int(answers['C0500']) != expected: 
            errors.append(f'C0500={answers["C0500"]} but C0200+C0300+C0400={expected}') 
    # PHQ-2 gate 
    a1 = int(answers.get('D0150A1', 0)) 
    b1 = int(answers.get('D0150B1', 0)) 



    if a1 + b1 < 3: 
        for letter in 'CDEFGHI': 
            for col in ['1','2']: 
                if answers.get(f'D0150{letter}{col}') is not None: 
                    errors.append(f'D0150{letter}{col} must be null on negative PHQ-2 
screen') 
        a2 = int(answers.get('D0150A2', 0)) if a1 == 1 else 0 
        b2 = int(answers.get('D0150B2', 0)) if b1 == 1 else 0 
        expected_d0160 = a2 + b2 
        if str(answers.get('D0160')) != str(expected_d0160): 
            errors.append(f'D0160={answers["D0160"]} but should be {expected_d0160}') 
    return errors

has issues — specifically A1010 field misuse and PHQ_MOOD_INTERVIEW being written into the gold standard. 
Those will be addressed separately in the next revision.
